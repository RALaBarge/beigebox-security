4-Tier Router

✦ Router

Send requests to any backend. Ollama locally, OpenRouter remotely, your own API. Automatic selection based on latency, availability, and model.

✧ Cache

Semantic deduplication. Understand that "What is AI?" and "Define artificial intelligence" are the same. Cache at the semantic level.

✦ Operator

Multi-turn tool execution. ReAct loops. Complex workflows. Full orchestration for reasoning-intensive tasks with artifact storage.

✧ Observe

Tap event logging. Real-time metrics. Understand every request, every token, every decision. Live dashboard. Cost tracking.

✦ Protect

Input/output filtering. Policy enforcement. Custom hooks. Compliance and guardrails built into every request.

✧ Configure

Web Studio for routing rules, model selection, and tool management. Live dashboard. No restarts needed for most changes.

How It Works

z: router=ollama, model=qwen3:32b User request → Z-command override → Static rules → Embedding classifier → Decision LLM → Selected backend → Cached response

Every request flows through our intelligent 4-tier pipeline. Z-commands override routing instantly. Static rules handle common cases. Embeddings classify simple queries. The decision LLM makes tough calls. All while caching and logging.
