# BeigeBox V3 — what changed and why

V3 is a security-focused refactor of V2. This file is the single source of truth for what was deleted, what was added, and the reasoning. If you're moving an integration from V2 to V3, read this end-to-end before changing your client.

V2 lives on at [`RALaBarge/beigebox`](https://github.com/RALaBarge/beigebox), `main` branch frozen.
V3 lives here at [`RALaBarge/beigebox-security`](https://github.com/RALaBarge/beigebox-security), `main` branch active.

---

## The thesis

V2 grew into a hybrid: thin proxy + agent harness + decision LLM + embedding classifier + multiple validation systems. Two reviews (the 2026-04-29 four-model panel and the 2026-05-01 grok ones in `docs/grok_reviews_2026_05_01.md`) converged on the same finding: the parts that made BeigeBox interesting were the *proxy and the observability*, and the agentic parts were either dead, unmaintained, or actively introducing attack surface that didn't match the project's "single-tenant defensive proxy" stance.

V3's rule: **if a feature requires "trust me" assumptions about untrusted input, it doesn't belong in a security-focused proxy.** Either close the input gap or delete the feature. Most got deleted.

The constitution that fell out of this is in [BEIGEBOX_IS_NOT.md](./BEIGEBOX_IS_NOT.md). New features get checked against it before they land.

## Deletions (the big ones)

### Agent harness — gone

**What was deleted:** Operator class, `operator/run` MCP schema + dispatcher + factory plumbing, harness/orchestrate (Wiggam multi-officer planning, Ralph TDD loop, Ensemble voting), the `/api/v1/orchestrations` HTTP routes, the entire `harness` and `discovery`/`lab` UI tabs and all their JS handlers (~3,000 lines).

**Why:** SOTA-class models running through BeigeBox can drive themselves via MCP. The agent loop belongs in the *driving client* — Claude Code, jcode, your custom SDK script — not in the proxy. Two losses if we keep an in-tree agent: (1) every agent improvement happens twice (once upstream, once here), (2) the in-proxy agent has the proxy's privileges by construction, which is exactly the threat-model boundary a security proxy should NOT cross.

**Migration:** point your agent client at `/mcp` for tools and `/v1/chat/completions` for inference. The client owns the loop.

### Decision LLM + embedding classifier + z-commands — gone

**What was deleted:** The four-tier hybrid routing system (literal-match → embedding-classifier → tier-2 router → decision-LLM judge), the `z: search`, `z: code`, `z: complex`, `z: help` user-input routing prefixes, the centroid rebuild endpoint, the classifier UI rebuild button.

**Why:** It was a clever solution to a problem that newer model APIs don't have anymore. Modern frontier models route their own queries inside the response. The hybrid layer added latency to every request, required a separate model deployment, and the classifier centroids needed periodic rebuilds nobody was doing. Deletion turned 5 round-trips into 1.

**Migration:** if you were using `z:` prefixes, drop them. The model picks its own approach.

### `python_interpreter` and `workspace_file` tools — gone

**Why:** Code execution and filesystem I/O are jobs for the *driving client*, not the proxy. The driving client (Claude Code, jcode, your script) already has its own sandbox / FS surface that the user explicitly authorized. Adding a second one inside BeigeBox just gave the LLM two ways to do the same thing, with the BeigeBox path having weaker isolation.

**Migration:** use your driving client's native code/file tools.

### Council + Voice — gone from the bundled UI

**What was deleted:** Council subtab (multi-model proposer/voter pattern), Voice subtab (push-to-talk + TTS), and all their backend endpoints (`/api/v1/council/propose`, voice WebRTC routes).

**Why:** Council was a proof-of-concept for ensemble voting before the harness module subsumed it. Voice was a research path that never paid off (the latency budget for voice loops is incompatible with proxying through cloud models). Both UIs accumulated half-finished JS that crashed the Chat tab on init via a leaked `opSel` reference.

**Migration:** for ensemble use cases, run multiple `/v1/chat/completions` from your client and reduce in-client.

### `SemanticCache` — gone

**Why:** The cache lookup added a vector query to every request even when the embedding wasn't needed, the hit rate in practice was below 5%, and stale cached responses were a real source of "the model said the wrong thing" bugs that took hours to diagnose.

**What survives:** `ToolResultCache` (deterministic tool results — calculator, datetime, etc.) is kept. That cache is correctness-safe by construction.

### `SQLiteStore` — gone (replaced by repos on a `BaseDB` shim)

**What was deleted:** The 1,800 LOC monolithic `storage/sqlite_store.py` with 7 dead tables and 21 unused methods.

**What replaced it:** Five focused repos (`ApiKeyRepo`, `ConversationRepo`, `QuarantineRepo`, `UserRepo`, `WireEventRepo`) on top of a thin `BaseDB` shim. Each repo owns its own schema and queries. Construction goes through `make_*_repo(db)` factories from `storage/repos/__init__.py`.

**Migration:** if you were importing from `beigebox.storage.sqlite_store`, switch to `beigebox.storage.repos`.

### `chromadb` vector backend — gone (postgres only)

**Why:** Two security concerns: (1) chromadb's default deployment writes plaintext content to disk, which is a PII leak waiting to happen; (2) the embedding metadata model made it hard to add the per-document quarantine signal needed for RAG-poisoning defense.

**What replaced it:** Postgres + pgvector. `vector_backend: postgres` in `config.yaml`. The connection string and pool size are configurable; `psycopg2-binary` + `pgvector` python package are required.

**Migration:** `pip install psycopg2-binary pgvector`, `apt install postgresql postgresql-17-pgvector`, `createdb beigebox`, `CREATE EXTENSION vector;`, then update `config.yaml`. See README quick-start.

### `SimplePasswordAuth` — gone (multi-key only)

**What was deleted:** The single-shared-password auth path (`POST /login`, the `/api/v1/auth/password` endpoints, the session-cookie code).

**What replaced it:** The multi-key registry (already existed) is now the only auth path. Plus an `auth.enabled: false` top-level kill switch in `runtime_config.yaml` for local-dev / single-user-trusted scenarios.

**Migration:** generate API keys via `bb auth keys add`. Bearer token in the `Authorization` header.

### `mcp_parameter_validator.py` — gone

**Why:** 924 LOC of validator code that was never instantiated outside its own docstring. Re-exported from `beigebox/security/__init__.py` (`ParameterValidator`, `MCPValidationResult`, `ValidationIssue`) but the only `ParameterValidator(...)` construction site outside the dead module was `tools/registry.py:57` which imports the LIVE one from `beigebox.tools.validation`. Pure ghost code — the same constitution/code mismatch as `python_interpreter`/`workspace_file`.

### `AMF` mesh subsystem — gone

**Why:** The Go-based agent mesh fabric (NATS event bus + mDNS discovery) was an experiment in distributed multi-agent coordination that didn't pan out. With the harness deletion, it had no remaining consumers.

### Other cleanups

- **Querystring auth (`?api_key=...`) removed** — leaks to logs and Referer headers; only `Authorization: Bearer` is accepted now.
- **Dead `2600-magazine` staging auto-ingest** — removed from `main.py`.
- **11 dead log-event helpers** — removed (see `513aacd`).
- **`docker/.mcp.json`** — Playwright MCP config removed (no longer wired).
- **Operator hook plumbing** — replaced by the generic `HookManager` (which already existed and now serves all use cases).

## Additions

### `Tool` Protocol contract

A runtime-checkable Protocol at `beigebox/tools/base.py`:

```python
@runtime_checkable
class Tool(Protocol):
    description: str
    def run(self, input_text: str) -> str | dict[str, Any]: ...
```

`ToolRegistry` is now typed `dict[str, Tool]` and runs an `isinstance(tool, Tool)` sanity check at startup. Contract violations raise at registry init instead of failing at first dispatch — the H-7 commit caught five plugin tools (`DiceTool`, `DocParserTool`, `UnitsTool`, `WiretapSummaryTool`, `ZipInspectorTool`) missing `description` that had been silently broken in the MCP `tools/list` response for months.

### Generic factory pattern (now enforced)

Every external interface goes through:

```
beigebox/<concern>/
  base.py          # Abstract base / Protocol
  <impl_a>.py      # Concrete impl A
  <impl_b>.py      # Concrete impl B
  __init__.py      # make_<concern>(type, **kwargs) factory
                   # + lazy _REGISTRY (optional-dep tolerant)
                   # + build_<concern>_kwargs(cfg, ...) (config → kwargs)
  plugins/         # (optional) auto-discovered userland impls
```

Already converted: `storage/backends/` (vector storage), `backends/` (LLM providers), `storage/db/` (SQL shim), `web_auth.py` (`make_auth`), `storage/wire_sink.py` (`make_sink`). When something doesn't follow this shape today, it's a known asymmetry to fix — not a pattern to mimic.

### Capture pipeline (single chokepoint)

`beigebox/capture.py` is the **only** place chat-completion telemetry gets serialized. `CaptureFanout` takes one captured turn and fans it to `ConversationRepo` + `WireLog` + `VectorStore`. Before this, the same turn was being serialized in 3 different places with 3 slightly different shapes; now there's one shape and one writer per sink.

### Proxy package split (G-1, G-2, G-3)

`proxy.py` (1,779 LOC) became a 6-module package:

- `proxy/core.py` — orchestrator (~1,245 LOC)
- `proxy/request_helpers.py` — `extract_conversation_id`, `get_model`, `get_latest_user_message`, `is_synthetic`, `dedupe_consecutive_messages`
- `proxy/body_pipeline.py` — `inject_generation_params/model_options`, `apply_window_config`
- `proxy/model_listing.py` — `list_models`, `transform_model_names`
- `proxy/request_inspector.py` — `RequestInspector` + finish helper
- `proxy/__init__.py` — re-exports

Streaming invariants (TTFT timer, no-yield-in-helpers, capture state machine, CancelledError propagation) preserved.

### Security routes return `{enabled: false}` instead of 503

`/api/v1/security/{audit, audit/patterns, honeypots, injection/stats, rag/quarantine, extraction/sessions}` now return `200 OK` with `{enabled: false, ...}` when the optional backing module isn't initialized. 503 ("service unavailable, retry later") was misleading — these features are intentionally off in many deployments.

### UI cleanup

The bundled web UI now has **6 tabs** (was 8): Dashboard, Tap, Chat, Studio, Security, Toolbox. Harness and Lab/Discovery tabs are gone. ~2,400 lines removed from `index.html`. Audit done via a CDP crawler at `/mnt/media/beigebox-data/crawl_ui.py` (run inside the LXC) — re-runnable for future audits.

## What survives

These survive intentionally and are not on the chopping block:

- **The bundled web UI itself.** Even when an external client (jcode, Warp, custom SDK) drives BeigeBox over `/v1` + `/mcp`, the bundled UI must remain self-sufficient — chat, wiretap viewer, config editor, toolbox editor. Multi-LLM features that survive (Studio playground) stay because they're what makes the web UI more capable than a single-model chat.

- **WASM runtime (`wasm_runtime.py`).** Looks dormant — the response-transform path lost its only writer when the routing decision LLM was deleted. Kept because the PDF input transform path (`transform_input("pdf_oxide", raw)` at the upload endpoint) is wired, and the broader bet is that the browser-as-OS future runs WASM modules. This is part of the interop story regardless of today's usage.

- **Pen/Sec MCP** (`/pen-mcp`). The 53-tool offensive-security wrapper surface stays. Different threat model from the main `/mcp` endpoint (operator-trusted, opt-in via config), built clean (argv-list `subprocess.run` only), authorization-gated on destructive tools.

- **All five storage repos.** Each has a clear owner and a stable contract.

- **The `HookManager` system.** Generic pre/post-request hooks. Was used by Operator (deleted) but the system itself is the right primitive for any future cross-cutting concern.

- **Skills (`beigebox/skills/`).** Each is MCP-discoverable, has its own `SKILL.md`, and follows the Tool Protocol contract.

## Open H-batch items (still queued)

A few items from the original H-batch are still open as of `50a0f9b`:

- **`tools/cdp.py`** — CLAUDE.md-flagged "we skipped the untrusted-input audit" leftover (Chrome cookie symlinks; subprocess paths derivable from model output). The "mimic mode" path inside cdp.py is confirmed broken by user testing. Currently parked — needs a delete-vs-factory-isolate audit before any change.
- **`tools/network_audit.py`** — audited 2026-05-03, verdict clean. No `shell=True`; argv-list everywhere; timeout int-coerced; IP strings validated upstream. No code change required, but the audit notes are worth carrying forward.
- **`PostgresWireSink AUTOINCREMENT`** — the postgres wire sink emits SQLite DDL and falls back to jsonl+sqlite. Real bug, not blocking.
- **Embedding preload uses Ollama-style `/api/embed`** — sends to OpenRouter which doesn't have that endpoint. Falls through silently after retries. Won't be fixed until an Ollama-shape embedding source is configured (which the canonical LXC deployment does).
- **Service user (not root) inside the LXC** — defense-in-depth nice-to-have. The LXC IS the isolation boundary so this is marginal, but worth doing.
