Sits in the middle.
Transparent to both sides.

Any OpenAI-compatible client — Open WebUI, LM Studio, Cursor, Aider, Continue, or your own app — points at BeigeBox instead of your model provider. BeigeBox handles the rest.

Requests flow through a deterministic 7-stage pipeline: parse → cache check → classify & route → inject context → forward → transform → log. Each stage is independently configurable and hot-reloadable.

The backend sees a plain OpenAI-compatible request. Your client sees an OpenAI-compatible response. BeigeBox does everything in between without either side needing to change.

What's in the box

Capabilities that come enabled or are one config flag away.

⇄

Smart routing

4-tier pipeline: Z-command overrides → keyword scorer → embedding classifier (~50 ms) → decision LLM. Session-sticky so a conversation stays on one backend.

◈

Semantic cache

Responses cached by meaning via cosine similarity, not by string or TTL. Near-identical queries return instantly — no backend round-trip, no model load.

⟳

Multi-backend router

Ollama, OpenRouter, or any OpenAI-compatible endpoint. Rolling P95 latency tracker auto-deprioritises slow backends. A/B traffic splitting with configurable weights.

◎

Full observability

Structured JSONL wire log, SQLite conversation history, per-model TTFT and P50/P90/P95/P99 latency, GPU layer stats, cost tracking, Grafana dashboards.

⊞

Orchestration modes

Side-by-side harness comparison. Ensemble voting with LLM arbiter + reasoning. Goal-driven Ralph loop: plan → dispatch → test → iterate via SSE stream.

⬡

MCP server

Model Context Protocol at POST /mcp. Exposes the full tool registry to Claude Desktop, external agents, or any MCP client. Progressive disclosure to avoid context flooding.

⬡

WASM transforms

Drop a WASI-compiled .wasm into wasm_modules/. BeigeBox buffers the stream, pipes it stdin→stdout, re-emits. Any language, zero Python deps. Timeout-safe.

⚙

Hot-reload config

Two-file design. config.yaml loaded once at startup. runtime_config.yaml mtime-checked per request — change model, temperature, toggles without restarting.

◑

Browser automation

Operator agent controls Chrome via DevTools Protocol. Navigate, screenshot, click, read DOM, type. Full I/O captured in context. Enabled via the CDP Docker Compose profile.

⬒

Voice I/O

Whisper (STT) + Kokoro (TTS) via the voice Docker Compose profile. Push-to-talk with configurable hotkey, auto-play TTS after each response. No changes to core proxy.

⊕

Plugins & hooks

Drop a *Tool class in plugins/ to auto-register. Pre/post request hooks for prompt injection detection, RAG injection, custom policy. Egress hooks for external metrics.

▤

Local-first storage

SQLite for conversations, metrics, cost. ChromaDB embedded (no separate service) for vectors and semantic memory. Portable files — no cloud dependency, no vendor lock-in.

4-tier routing pipeline

Every request cascades through this sequence. Simpler requests short-circuit early — only borderline cases reach the decision LLM.

Tier 1

Z-command overrides

User types z: complex or z: code as a message prefix. Parsed before anything else. Zero compute, immediate dispatch.

cost: ~0 ms
Tier 2

Keyword scorer

Agentic keyword pattern matching. Catches obvious code / tool / complex signals without embedding. Exits early for clear-cut cases.

cost: ~1 ms
Tier 3

Embedding classifier

Embeds the query with nomic-embed-text and checks cosine similarity against pre-built centroids per tier. Covers the bulk of requests reliably.

cost: ~50 ms
Tier 4

Decision LLM

A small, fast model (e.g. llama3.2:3b) judges ambiguous cases that didn't classify cleanly. Optional — disable to cap at tier 3.

cost: ~200–800 ms

Routing decisions are session-sticky: the first classification for a conversation is reused for all subsequent turns, preventing mid-conversation backend flips.

Z-commands

Prefix any message with z: to override routing, invoke tools, or branch the conversation inline — no UI needed.

z: simpleRoute to the fast / cheap model
z: complexRoute to the large / capable model
z: codeRoute to the code-specialist model

Quick start

Full stack — Ollama + BeigeBox — running in under 2 minutes via Docker Compose.

Docker Compose (recommended)

Pulls Ollama, starts BeigeBox, wires everything up. GPU passthrough configured via .env.

# clone with submodules
git clone --recursive \
  https://github.com/ralabarge/beigebox.git
cd beigebox/docker
cp env.example .env
docker compose up -d

Web UI → http://localhost:1337
API endpoint → http://localhost:1337/v1

Dev mode (bare metal)

Editable install with hot-reload. Changes to runtime_config.yaml take effect on next request — no restart.

pip install -e .
cp config.example.yaml config.yaml
# edit config.yaml — set your backend URL
beigebox dial

FastAPI at localhost:8000 · auto-docs at /docs

Any OpenAI-compatible client works out of the box. Point Open WebUI, LM Studio, Cursor, Aider, or Continue at http://localhost:1337/v1 — no code changes required.

Security by default

Designed on the assumption that supply chain compromise is inevitable. Three independent defense layers.

Layer 1 · Prevention

Hash-locked dependencies

3,280 pip hashes. Pinned Docker image digests. CVE scanning on every build via pip-audit, bandit, semgrep, gitleaks, and trivy.

Layer 2 · Containment

Read-only root, dropped capabilities

Docker runs read-only root filesystem, network segmentation, capability drop, and an unprivileged user. Compromised code cannot persist to disk.

Layer 3 · Detection

Full Tap wire logging (18+ event types), metrics, and automated git hooks. Anomalies surface in the dashboard within 0.1s of occurrence.
