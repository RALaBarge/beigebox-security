## Key Decisions

- **BrowserBox is primary, CDP is fallback.** BrowserBox already exists, has 37 tools, carries session cookies, has BeigeBox integration spec in its README. CDP/Playwright are for headless/CI testing only.
- **Anti-MCP stance validated.** HN practitioner consensus: CLI + skill docs > MCP. Context window cost of MCP is the killer problem.
- **Level 2 lazy loading** is the recommended BrowserBox integration: single `browser` meta-tool in context (~200 tokens), full schemas loaded on demand.
- **Tool profiles by model tier**: minimal (5 tools for 3B), standard (12 for 7B-13B), full (30+ for 70B+/API).
